# Search-Download Orphan Fix — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Prevent search/artist-profile downloads from being orphan-swept on next app launch by linking each new download to a protected "Your Downloads" playlist that renders in the Home Stash Mixes (beta) section.

**Architecture:** Single new `PlaylistType.DOWNLOADS_MIX` enum value. One seeded playlist per install, idempotent at startup. `TrackActionsDelegate.handleDownloadSuccess` links each new track to it. Home Stash Mixes filter widens to include both types. "Remove from playlist" on a `DOWNLOADS_MIX` track performs a full track delete (since it has no other home). No schema migration, no backfill, no Library row.

**Tech Stack:** Kotlin, Android (Compose), Room, Hilt, JUnit 4, Mockk, Kotlin coroutines.

**Spec:** `docs/superpowers/specs/2026-04-24-search-download-orphan-fix-design.md`

```kotlin
override suspend fun ensureDownloadsMixSeeded(): Long {
    val existing = playlistDao.findBySourceId(DOWNLOADS_MIX_SOURCE_ID)
    if (existing != null) return existing.id
    val entity = PlaylistEntity(
        name = "Your Downloads",
        source = MusicSource.BOTH,
        sourceId = DOWNLOADS_MIX_SOURCE_ID,
        type = PlaylistType.DOWNLOADS_MIX,
        syncEnabled = false,
        dateAdded = Instant.now(),
    )
    return playlistDao.insert(entity)
}

override suspend fun linkTrackToDownloadsMix(trackId: Long) {
    val playlistId = ensureDownloadsMixSeeded()
    playlistTrackCrossRefDao.insertOrIgnore(
        PlaylistTrackCrossRef(playlistId = playlistId, trackId = trackId),
    )
}

companion object {
    private const val DOWNLOADS_MIX_SOURCE_ID = "stash_downloads_mix"
}
```

## Risks & rollback

- **If `ensureDownloadsMixSeeded` fails on a device** (e.g., DAO exception): the track is still inserted by `insertTrack`, just without the link. On next launch the orphan sweep would delete it — same pre-fix behavior. Degraded but not catastrophic. Surface any crash via standard crash reporting.
- **If a user somehow ends up with two DOWNLOADS_MIX playlists** (e.g., manual DB edit, partial migration): the unique `source_id` index on `playlists` prevents this. If it ever happens, the seeder's `findBySourceId` returns the first; the second exists but is unreachable by the seeder path. Manual DB cleanup only.
- **Rollback:** revert `fix: 0.6.2`. Users who upgraded will retain the seeded playlist row in their DB — harmless; it just renders as an empty `DOWNLOADS_MIX` playlist the next version's UI doesn't filter for. Optional cleanup migration in a future release.
